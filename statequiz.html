State Management Quiz

1. Can post requests be bookmarked? What are the problems?

No, Only get request can be bookmarked. The problem is we can't send post parameters

2. What is the purpose of request dispatching?

Defines an object that receives requests from the clientand sends them to any resource (such as a servlet,HTML file, or JSP file) on the server. The servletcontainer creates the RequestDispatcher object,which is used as a wrapper around a server resource locatedat a particular path or given by a particular name.

3. What is the difference between redirect and request dispatch?

Redirect is sends a temporary redirect response to the client

Request dispatch sends to anyresource

4. What is an attribute ?

An object bound into one of the three servlet API objects:

  • HttpServletRequest
  • HttpSession
  • ServletContext

5. What is the difference between attributes and parameters?

Parameters are from client side. Attributes are for server-side usage only.

6. What are dangers of using attributes?

Context scope attributes are like nuclear weapon. BEcause it's shared by every servlet and every request in application

7. What does it mean to say that http is stateless? Give an example of a stateful protocol.

Http is stateless. It means that after a web server responds to a request, there is no information of the request or who sent it. FTP (File Transfer Protocol) and Telnet are stateful protocol.

8. Give 5 different methods for maintaining state information (count each attribute scope as one method).

  1. request scope: destroyed when servlet finishes processing request
  2. session scope: destroyed when user closes browser
  3. application scope destroyed when Container stopped.
  4. Cookies saved on browser: response.addCookie
  5. hidden fields on form: input type="hidden" name="hiddenState"

11. What is the purpose of URL rewriting?

If your browser does not support cookies, URL rewriting provides you with another session tracking alternative. URL rewriting is a method in which the requested URL is modified to include a session ID.
